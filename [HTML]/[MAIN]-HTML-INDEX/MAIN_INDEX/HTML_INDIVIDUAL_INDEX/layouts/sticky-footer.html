Sticky Footer Layout

A sticky footer stays at the bottom of the viewport even when there's minimal content on the page. When content is longer than the viewport, the footer naturally moves down with the content.

How It Works

This layout uses modern Flexbox to achieve the sticky footer effect. Here's the key technique:

body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.main {
  flex: 1; /* Grows to fill available space */
}

Why Use This Pattern?

  • Prevents awkward floating footers on pages with little content
  • Creates a more polished, professional appearance
  • Works consistently across all modern browsers
  • Requires minimal CSS and no JavaScript
  • Responsive and mobile-friendly

Alternative Methods

While Flexbox is the recommended modern approach, here are other methods you might encounter:

Method 2: CSS Grid

body {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

Method 3: Calc (Old-school)

.main {
  min-height: calc(100vh - 200px); /* Subtract header + footer height */
}

Common Use Cases

  • Login/Signup pages (typically short content)
  • Error pages (404, 500, etc.)
  • Landing pages with minimal content
  • Any page where content length varies
  • Single-page applications

Try it out: Even though this page has moderate content, the footer will always stay at the bottom. If you remove most of the text, the footer won't float up into the middle of the page.
