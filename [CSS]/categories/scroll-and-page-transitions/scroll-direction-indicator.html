Scroll Direction Indicator

Visual feedback showing whether user is scrolling up or down

1 Arrow Indicator

Simple arrow that rotates based on scroll direction with color coding

Direction Awareness

Knowing which way the user is scrolling can trigger different UI behaviors - like showing or hiding a header.

This simple arrow indicator changes direction and color based on scroll movement. Cyan for down, pink for up.

Use Cases

Navigation bars that hide on scroll down and reappear on scroll up. Floating action buttons that move out of the way.

Progress indicators that emphasize forward movement. Contextual hints that adapt to user behavior.

Implementation

Track the previous scroll position and compare it to the current. The difference determines direction.

A small threshold prevents flickering when the user pauses scrolling.

Visual Design

Color coding makes direction immediately obvious. The circular container provides a clean, focused indicator.

/* Track scroll direction */ let lastScrollTop = 0; container.addEventListener('scroll', () => { const st = container.scrollTop; const direction = st > lastScrollTop ? 'down' : 'up'; lastScrollTop = st; });

2 Animated Chevron

Bouncing chevron that suggests continued scrolling in that direction

Motion Guides the Eye

The bouncing animation draws attention to the scroll direction and subtly encourages continued scrolling.

The chevron shape is universally understood as a directional indicator.

Animation Psychology

Rhythmic motion creates expectation. Users unconsciously anticipate the next bounce, keeping them engaged.

The animation direction matches the scroll direction, reinforcing the user's action.

When to Use

Landing pages that want users to continue scrolling. Onboarding flows guiding users through content.

Long-form content where engagement is key. Interactive stories or presentations.

Accessibility

For users with motion sensitivity, consider a prefers-reduced-motion media query to disable the bounce.

/* Bouncing animation */ @keyframes chevronBounce { 0%, 100% { transform: translateY(-50%); } 50% { transform: translateY(calc(-50% + 5px)); } } .direction-chevron--animated { animation: chevronBounce 1.5s ease-in-out infinite; }

3 Text Label

Explicit text indicating "Scrolling Up" or "Scrolling Down"

Clarity Over Style

Sometimes explicit is better than implicit. A text label leaves no room for misinterpretation.

Combined with an arrow icon, users get both visual and textual confirmation.

Accessibility Benefits

Screen readers can announce the direction. Users with cognitive differences may find text clearer than symbols.

The label also serves as documentation for developers inspecting the page.

Localization

Text labels need translation. Consider using CSS content property or data attributes for easy updates.

Some languages may need different layouts - RTL support is important.

Design Considerations

Keep the label compact. All-caps text with letter-spacing creates a technical, dashboard feel.

↓ Scrolling Down
/* Text label styles */ .direction-label { font-size: 0.75rem; text-transform: uppercase; letter-spacing: 1px; }

4 Color-Coded Bar

Edge bar that glows different colors based on direction

Ambient Indication

This subtle edge glow provides direction feedback without adding UI elements to the content area.

The gradient fades towards the scroll direction, creating a natural visual flow.

Peripheral Vision

Users perceive the color change in their peripheral vision. It's noticed without demanding direct attention.

This makes it ideal for reading-heavy applications where focus should remain on content.

Color Semantics

Cyan (cool) for down suggests calm, forward progress. Pink (warm) for up suggests return, going back.

Neutral gray when idle tells users scrolling has stopped.

Implementation

A single div with gradient background that changes via CSS class toggle. Lightweight and performant.

/* Gradient bar */ .direction-bar--down { background: linear-gradient(to bottom, transparent, cyan); } .direction-bar--up { background: linear-gradient(to top, transparent, pink); }

5 Hide on Idle

Indicator fades in during scroll, fades out when stopped

Show Only When Relevant

Why show a scroll indicator when the user isn't scrolling? This version appears only during active scrolling.

After a brief pause, it fades out to reduce visual clutter.

Reduced Visual Noise

Modern interfaces can feel overwhelming. Showing UI elements only when needed creates a cleaner experience.

The fade transition is gentle - appearing quickly, disappearing slowly.

Technical Implementation

Use a timeout that resets on each scroll event. When it fires, hide the indicator.

The timeout duration (usually 1-2 seconds) balances responsiveness with calmness.

Use Cases

Reading apps, documentation viewers, or any interface where content is the star.

/* Show on scroll, hide on idle */ let hideTimeout; container.addEventListener('scroll', () => { indicator.classList.add('visible'); clearTimeout(hideTimeout); hideTimeout = setTimeout(() => { indicator.classList.remove('visible'); }, 1500); });

6 Dual Indicator

Both up and down arrows visible, active one is highlighted

Always-Visible Context

With both arrows always visible, users can see which direction is currently active without remembering the previous state.

The inactive arrow remains dimmed, providing context without distraction.

Click to Navigate

These arrows could double as navigation - click up to scroll to top, click down to scroll to bottom.

Adding this interactivity increases the utility of the indicator.

State Awareness

At the top of content, the up arrow might be disabled. At the bottom, the down arrow.

This provides additional context about scroll boundaries.

Design Variations

Could be positioned vertically (as shown), horizontally, or even as a connected toggle switch.

/* Dual indicator - highlight active */ .dual-arrow { opacity: 0.5; } .dual-arrow.active { opacity: 1; border-color: var(--jazer-cyan); }

7 Velocity Indicator

Shows both direction and speed of scrolling

Speed Matters

Sometimes knowing how fast the user is scrolling is as important as knowing the direction.

Fast scrolling might indicate skimming, slow scrolling suggests careful reading.

Visual Representation

The bar fills from center outward. The more it fills, the faster the scroll. Direction determines which half fills.

The center dot represents the neutral "stopped" state.

Analytics Insight

This visual can be backed by actual velocity tracking. Log reading patterns for content optimization.

Identify which sections users skim vs. read carefully.

Adaptive UI

Fast scrolling could trigger a "scroll to top" button. Slow scrolling could enable article highlights.

Velocity
/* Calculate scroll velocity */ const velocity = Math.abs(currentPos - lastPos); const normalized = Math.min(velocity / 50, 1); // 0-1 fill.style.height = (normalized * 50) + '%';
